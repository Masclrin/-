# 宏执行框架说明

这是一个 **Windows平台下的高精度宏执行框架** ，主要应用于游戏辅助和自动化操作场景。

- `宏执行框架2.1.py`：主程序，负责监听触发键、加载/编译/执行 JSON 宏，包含延迟补偿等逻辑。
- `precision_engine_v5.py`：高精度睡眠与忙等实现（使用 WinAPI + 可选本地机器码忙等，降低延迟抖动）。
- `interception_input.py`：直接调用 Interception 的 DLL，实现内核级输入注入（若驱动已安装）。

## 1. 当前推荐版本

- 推荐日常使用：`宏执行框架2.1.py`
- 备选版本：
  - `宏执行框架1.94.py`（PID 稳定性增强版，长宏效果更佳。需长期调参）
  - `宏执行框架2.0.py`（已支持全局暂停与按键复用控制，PI退回至1.8版）

## 2. 快速启动

1. 安装依赖：双击 `1_安装依赖.bat`
2. 安装驱动（管理员）：双击 `2_安装驱动_管理员运行.bat`
3. 重启系统（驱动生效必需）
4. 启动脚本：
   - 方式 A：右键，管理员启动 `3_启动宏.bat`
   - 方式 B：管理员模式手动执行 `python "宏执行框架2.1.py"`

## 3. 宏格式（通用）

宏文件是 JSON 数组，常用动作：

- `kd` / `ku`：按键按下与抬起
- `md` / `mu`：鼠标按下与抬起
- `view`：鼠标视角平滑移动  `["view", [dx, dy], <dur_ms>]`
- `wait`：纯等待
- `loop`：循环块 `["loop", <count>, [<block>]]`
- `import`：导入子宏 `["import", "subfile.json"]`
- 示例：`[ ["md","left",0], ["mu","left",120], ,["wait",200], ["loop",3, [["kd","q",0],["ku","q",50]]] ]`

## 4. 常见问题

- 提示 `Interception 不可用，回退 SendInput`：检查驱动安装是否成功并已重启。
- 启动后无动作：检查 `MACRO_BINDINGS` 指向的宏文件是否存在。
- 触发异常频繁：检查 `MACRO_RUNTIME_SETTINGS` 的 `execution_mode`、`stop_mode` 与 `running_press_mode`。

## 5. 风险提示

- 内核驱动与输入注入在部分场景可能触发风控或反作弊检测！

## 6. 版本更新记录（从 1.8 开始）

说明：`1.8` 为历史基线版本，以下为后续每个新脚本版本的改动要点。

### 1.9（`宏执行框架1.9 copy.py`）

- 于interception中修复了吞左键的问题，修复中文乱码问题。
- 增加测试追踪开关与执行明细输出配置。

### 1.91（`宏执行框架1.91.py`）

- 加入更激进的系统调度尝试：REALTIME 提权与 Pro Audio 调度注入。
- 将延迟补偿参数调至“零误差倾向”实验配置。
- 强化测试报表输出定位（默认 final 汇总）。

### 1.92（`宏执行框架1.92.py`）

- 保留1.91版本中有效优化项,将提权/调度能力改为可开关（`ENABLE_REALTIME_PRIORITY`、`ENABLE_PRO_AUDIO`）。
- 调整 PI 参数到更稳健区间，降低 1.91 的激进抖动风险。
- 提供实时/汇总两种测试输出模式。

### 1.93_alpha（`宏执行框架1.93_alpha.py`）

- 接入 `precision_engine_v5b`，引入智能控制器接口。
- 新增控制器模式切换思路（legacy/smart/ultra 的实验方向）。
- 为控制策略验证分支。

### 1.93_beta 自适应 PI（`宏执行框架1.93_beta自适应PI.py`）

- 引入自适应 PI：按误差区间动态放大/收缩增益。
- 加入振荡检测、振荡衰减与软启动机制。
- 聚焦“误差收敛速度”与“超调抑制”的平衡测试。

### 1.94（`宏执行框架1.94.py`）

- 从 PI 升级到 PID（新增 `LAG_KD`）。
- 增加积分非对称限幅、积分斜率限制、负补偿硬限制。
- 重点修正积分突变和震荡，提升长宏稳定性。

### 2.0（`宏执行框架2.0.py`）

- 增加运行态交互控制：
  - 全局暂停键 `F8`
  - 增加了不同执行模式时，再次按下或抬起的宏暂停与继续功能
- 新增“PI/前馈”双控制补偿路线开关：
  - `LAG_USE_PI_CONTROLLER`
  - `LAG_USE_FEEDFORWARD`
- 引入前馈控制参数组（窗口、限幅、EMA、过零抑制等）。

### 2.1（`宏执行框架2.1.py`）

- 扩展按压行为模式：新增 `hold_pause`（按住暂停、抬起继续）。
- 默认策略转向前馈补偿（`LAG_USE_FEEDFORWARD=True`、`LAG_USE_PI_CONTROLLER=False`）。
- 修复了暂停后延迟误差显示过大的问题
- 调整输出与运行默认值，降低实时打印对时序精度的干扰。
